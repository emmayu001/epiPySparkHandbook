Introduction

Welcome to The Epidemiologist PySpark Handbook, a comprehensive guide designed specifically for epidemiologists and public health professionals working with large-scale health data.

📊 Why PySpark for Epidemiology? Traditional tools like R and Stata excel with datasets under 10GB, but modern epidemiological datasets—EHRs, genomic data, claims databases, and population-level surveillance systems—often exceed terabytes. PySpark enables you to analyze these massive datasets efficiently using distributed computing.
🚀

Scalability

Process datasets from gigabytes to petabytes without changing your code

🔄

Familiar Syntax

SQL-like operations and dataframe syntax similar to pandas and dplyr

⚡

Speed

In-memory computing delivers 100x faster analytics than traditional disk-based systems

Who This Handbook Is For

  • Epidemiologists working with large healthcare databases (claims, EHR, registries)
  • Public health researchers analyzing population-level surveillance data
  • Data scientists in health agencies processing massive datasets
  • R/Stata users transitioning to big data platforms

Prerequisites

This handbook assumes you have:

  • Basic programming knowledge (Python preferred, but R experience translates well)
  • Fundamental epidemiological concepts (study designs, measures of association, confounding)
  • Familiarity with data manipulation (filtering, grouping, joining)
💡 Learning Approach Each chapter includes practical examples using real-world epidemiological scenarios. Code snippets are provided with explanations of both the technical implementation and epidemiological interpretation.

Creating a Spark Session

Every PySpark application starts with a SparkSession:

from pyspark.sql import SparkSession

# Create Spark session with configuration
spark = SparkSession.builder \
    .appName("EpidemiologyAnalysis") \
    .config("spark.driver.memory", "4g") \
    .config("spark.executor.memory", "4g") \
    .getOrCreate()

# Check Spark version and config
print(f"Spark version: {spark.version}")
print(f"Master: {spark.sparkContext.master}")
⚠️ Memory Configuration Adjust memory settings based on your dataset size and available RAM. For datasets under 50GB on a laptop, 4-8GB driver memory is usually sufficient.

PySpark Basics

DataFrames: The Core Abstraction

PySpark DataFrames are distributed collections of data organized into named columns, similar to tables in SQL or dataframes in R/pandas.

# Create a DataFrame from data
from pyspark.sql import Row

data = [
    Row(patient_id=1, age=45, disease="COVID-19", outcome="Recovered"),
    Row(patient_id=2, age=67, disease="COVID-19", outcome="Deceased"),
    Row(patient_id=3, age=34, disease="COVID-19", outcome="Recovered")
]

df = spark.createDataFrame(data)
df.show()

Reading Data

PySpark supports various file formats common in epidemiology:

# CSV files (most common)
df = spark.read.csv("surveillance_data.csv", header=True, inferSchema=True)

# Parquet (optimized for big data)
df = spark.read.parquet("ehr_records.parquet")

# JSON
df = spark.read.json("api_response.json")

# Database connection
df = spark.read.jdbc(
    url="jdbc:postgresql://localhost:5432/health_db",
    table="patients",
    properties={"user": "analyst", "password": "secret"}
)

Basic Operations

Viewing Data

# Show first 20 rows
df.show()

# Show specific number of rows
df.show(5, truncate=False)

# Display schema
df.printSchema()

# Get column names
df.columns

# Count rows
df.count()

Selecting Columns

# Select specific columns
df.select("patient_id", "age", "disease").show()

# Select and rename
df.select(
    col("patient_id").alias("id"),
    col("age"),
    col("disease")
).show()

# Drop columns
df.drop("unnecessary_col").show()

Filtering Rows

from pyspark.sql.functions import col

# Simple filter
df.filter(col("age") > 50).show()

# Multiple conditions
df.filter(
    (col("age") > 50) & 
    (col("disease") == "COVID-19")
).show()

# Using SQL-like WHERE
df.where("age > 50 AND disease = 'COVID-19'").show()
💡 R to PySpark Translation
  • filter() in dplyr = filter() or where() in PySpark
  • select() in dplyr = select() in PySpark
  • mutate() in dplyr = withColumn() in PySpark
  • group_by() %>% summarise() = groupBy().agg()

Key Conceptual Differences

Common R Operations in PySpark

Data Manipulation

# R (dplyr)
# df %>%
#   filter(age > 50) %>%
#   select(patient_id, age, disease) %>%
#   mutate(age_group = case_when(
#     age < 18 ~ "Child",
#     age < 65 ~ "Adult",
#     TRUE ~ "Senior"
#   ))

# PySpark equivalent
from pyspark.sql.functions import when

df_transformed = df \
    .filter(col("age") > 50) \
    .select("patient_id", "age", "disease") \
    .withColumn("age_group",
        when(col("age") < 18, "Child")
        .when(col("age") < 65, "Adult")
        .otherwise("Senior")
    )

Aggregation

# R (dplyr)
# df %>%
#   group_by(disease) %>%
#   summarise(
#     n = n(),
#     mean_age = mean(age, na.rm=TRUE),
#     case_fatality = sum(outcome == "Deceased") / n()
#   )

# PySpark equivalent
from pyspark.sql.functions import count, mean, sum, when

df.groupBy("disease") \
    .agg(
        count("*").alias("n"),
        mean("age").alias("mean_age"),
        (sum(when(col("outcome") == "Deceased", 1).otherwise(0)) / count("*"))
            .alias("case_fatality")
    ).show()
📊 Why Lazy Evaluation? Unlike R which executes each line immediately, PySpark builds an execution plan and only runs it when you call an "action" like show(), count(), or write(). This allows Spark to optimize the entire workflow before execution—critical for performance with large datasets.

Importing Data

Reading Large CSV Files

# Basic CSV import
df = spark.read.csv(
    "surveillance_data.csv",
    header=True,
    inferSchema=True  # Automatically detect data types
)

# With explicit schema (faster for large files)
from pyspark.sql.types import StructType, StructField, StringType, IntegerType, DateType

schema = StructType([
    StructField("patient_id", StringType(), False),
    StructField("diagnosis_date", DateType(), True),
    StructField("age", IntegerType(), True),
    StructField("sex", StringType(), True),
    StructField("disease_code", StringType(), True)
])

df = spark.read.csv(
    "surveillance_data.csv",
    header=True,
    schema=schema
)

Reading Multiple Files

# Read all CSV files in a directory
df = spark.read.csv("data/year_2023/*.csv", header=True)

# Read with wildcard pattern
df = spark.read.csv("data/surveillance_202*.csv", header=True)

# Combine multiple directories
df = spark.read.csv(["data/2022/", "data/2023/"], header=True)

Working with Parquet

Parquet is the preferred format for big data—it's columnar, compressed, and preserves schema:

# Read Parquet
df = spark.read.parquet("ehr_data.parquet")

# Write to Parquet (saves space and loads faster)
df.write.parquet("processed_data.parquet", mode="overwrite")

# Partitioned writes (organizes by column values)
df.write.partitionBy("year", "disease") \
    .parquet("surveillance_partitioned.parquet")
⚠️ File Format Recommendations
  • CSV: Use for initial data receipt and sharing with non-technical users
  • Parquet: Use for all intermediate and final analytical datasets (10-100x smaller, 100x faster)
  • Delta Lake: Use for frequently updated tables requiring ACID transactions

Data Cleaning

Handling Missing Values

from pyspark.sql.functions import col, when, isnan, isnull

# Count missing values per column
df.select([
    count(when(col(c).isNull(), c)).alias(c)
    for c in df.columns
]).show()

# Drop rows with any null values
df_clean = df.dropna()

# Drop rows with nulls in specific columns
df_clean = df.dropna(subset=["patient_id", "diagnosis_date"])

# Fill missing values
df_filled = df.fillna({
    "age": 0,
    "sex": "Unknown",
    "bmi": df.agg({"bmi": "mean"}).collect()[0][0]  # Fill with mean
})

Detecting and Removing Duplicates

# Count total duplicates
print(f"Total rows: {df.count()}")
print(f"Distinct rows: {df.distinct().count()}")

# Remove exact duplicates
df_dedup = df.dropDuplicates()

# Remove duplicates based on specific columns (e.g., patient-date)
df_dedup = df.dropDuplicates(["patient_id", "visit_date"])

Data Type Conversion

from pyspark.sql.functions import to_date, to_timestamp
from pyspark.sql.types import IntegerType, DoubleType

# Convert string to date
df = df.withColumn("diagnosis_date", 
                   to_date(col("diagnosis_date"), "yyyy-MM-dd"))

# Convert string to integer
df = df.withColumn("age", col("age").cast(IntegerType()))

# Convert string to numeric (handles non-numeric as null)
df = df.withColumn("lab_value", col("lab_value").cast(DoubleType()))

String Cleaning

from pyspark.sql.functions import trim, upper, lower, regexp_replace

# Trim whitespace
df = df.withColumn("disease", trim(col("disease")))

# Standardize case
df = df.withColumn("disease", upper(col("disease")))

# Remove special characters
df = df.withColumn("patient_id",
                   regexp_replace(col("patient_id"), "[^0-9]", ""))

# Replace values
df = df.withColumn("sex",
    when(col("sex").isin("M", "Male", "male"), "Male")
    .when(col("sex").isin("F", "Female", "female"), "Female")
    .otherwise("Unknown")
)

Outlier Detection

from pyspark.sql.functions import percentile_approx

# Calculate quartiles for outlier detection
quantiles = df.approxQuantile("age", [0.25, 0.75], 0.01)
q1, q3 = quantiles[0], quantiles[1]
iqr = q3 - q1

# Flag outliers (values outside 1.5*IQR)
df = df.withColumn("age_outlier",
    when((col("age") < (q1 - 1.5 * iqr)) | 
         (col("age") > (q3 + 1.5 * iqr)), True)
    .otherwise(False)
)

# Count outliers
df.filter(col("age_outlier") == True).count()

Descriptive Statistics

Summary Statistics

# Quick summary of numeric columns
df.describe().show()

# Specific statistics
from pyspark.sql.functions import mean, stddev, min, max, count

df.select([
    count("patient_id").alias("n"),
    mean("age").alias("mean_age"),
    stddev("age").alias("sd_age"),
    min("age").alias("min_age"),
    max("age").alias("max_age")
]).show()

Frequency Tables

# One-way frequency table
df.groupBy("disease").count().orderBy("count", ascending=False).show()

# Two-way frequency table
df.groupBy("disease", "outcome") \
    .count() \
    .orderBy("disease", "outcome") \
    .show()

# With proportions
from pyspark.sql.window import Window

total = df.count()
df.groupBy("disease") \
    .agg(count("*").alias("n")) \
    .withColumn("proportion", col("n") / total) \
    .withColumn("percentage", col("proportion") * 100) \
    .show()

Crosstabulation

# Create contingency table
df.crosstab("exposure", "outcome").show()

# Chi-square test approximation (requires pivot)
contingency = df.groupBy("exposure") \
    .pivot("outcome") \
    .count() \
    .fillna(0)

contingency.show()
💡 Epidemiological Tip Always check your denominator! Use count() carefully—it counts non-null values. For person-time denominators, sum the follow-up time explicitly.

Calculating Rates & Ratios

Incidence Rate

# Calculate incidence rate (per 100,000 person-years)
from pyspark.sql.functions import sum

rates = df.groupBy("disease") \
    .agg(
        sum("cases").alias("total_cases"),
        sum("person_years").alias("total_person_years")
    ) \
    .withColumn("incidence_rate",
        (col("total_cases") / col("total_person_years")) * 100000
    )

rates.show()

Age-Standardized Rates

# Direct standardization
# Join with standard population
standard_pop = spark.read.csv("standard_population.csv", header=True)

age_specific = df.groupBy("age_group") \
    .agg(
        sum("cases").alias("cases"),
        sum("population").alias("pop")
    ) \
    .withColumn("rate", col("cases") / col("pop") * 100000)

# Join with standard population and calculate weighted rate
standardized = age_specific.join(standard_pop, "age_group") \
    .withColumn("weighted_cases", col("rate") * col("standard_pop")) \
    .agg(sum("weighted_cases").alias("age_standardized_rate"))

standardized.show()

Risk Ratios and Odds Ratios

# Calculate 2x2 table
exposed_cases = df.filter((col("exposed") == 1) & (col("case") == 1)).count()
exposed_controls = df.filter((col("exposed") == 1) & (col("case") == 0)).count()
unexposed_cases = df.filter((col("exposed") == 0) & (col("case") == 1)).count()
unexposed_controls = df.filter((col("exposed") == 0) & (col("case") == 0)).count()

# Risk Ratio
risk_exposed = exposed_cases / (exposed_cases + exposed_controls)
risk_unexposed = unexposed_cases / (unexposed_cases + unexposed_controls)
risk_ratio = risk_exposed / risk_unexposed

# Odds Ratio
or_value = (exposed_cases * unexposed_controls) / (exposed_controls * unexposed_cases)

print(f"Risk Ratio: {risk_ratio:.2f}")
print(f"Odds Ratio: {or_value:.2f}")

Confidence Intervals

import math

# 95% CI for proportions (Wald method)
def proportion_ci(cases, total, alpha=0.05):
    p = cases / total
    se = math.sqrt(p * (1 - p) / total)
    z = 1.96  # 95% CI
    ci_lower = p - z * se
    ci_upper = p + z * se
    return (ci_lower, ci_upper)

# Example usage
n_cases = 45
n_total = 1000
ci = proportion_ci(n_cases, n_total)
print(f"Proportion: {n_cases/n_total:.3f} (95% CI: {ci[0]:.3f}-{ci[1]:.3f})")

Regression Models

Logistic Regression

from pyspark.ml.feature import VectorAssembler
from pyspark.ml.classification import LogisticRegression

# Prepare features
assembler = VectorAssembler(
    inputCols=["age", "bmi", "smoking"],
    outputCol="features"
)

df_features = assembler.transform(df)

# Fit logistic regression
lr = LogisticRegression(
    featuresCol="features",
    labelCol="disease",
    maxIter=10,
    regParam=0.01
)

model = lr.fit(df_features)

# View coefficients (log odds ratios)
print("Coefficients:", model.coefficients)

# Get odds ratios
import numpy as np
odds_ratios = np.exp(model.coefficients.toArray())
print("Odds Ratios:", odds_ratios)

Performance Optimization

Caching

# Cache frequently accessed DataFrames in memory
df_filtered = df.filter(col("year") == 2023)
df_filtered.cache()  # or .persist()

# Use the cached DataFrame multiple times
df_filtered.groupBy("disease").count().show()
df_filtered.groupBy("age_group").count().show()

# Unpersist when done
df_filtered.unpersist()

Partitioning

# Repartition for better parallelism
df_repartitioned = df.repartition(100)  # 100 partitions

# Coalesce to reduce partitions (no shuffle)
df_coalesced = df.coalesce(10)

# Partition by column for efficient filtering
df.write.partitionBy("year", "state") \
    .parquet("surveillance_partitioned")

Broadcast Joins

from pyspark.sql.functions import broadcast

# For small lookup tables (<100MB), broadcast to all workers
small_ref_table = spark.read.csv("icd_codes.csv", header=True)

# Broadcast join (much faster than shuffle join)
result = df.join(broadcast(small_ref_table), "disease_code")

Query Optimization Tips

💡 Performance Best Practices
  • Filter early: Apply filters before joins and aggregations
  • Select only needed columns: Don't use select("*")
  • Use Parquet: Columnar format is 10-100x faster than CSV
  • Avoid collect(): Don't bring entire large DataFrames to driver
  • Monitor Spark UI: Check localhost:4040 for execution plans
  • Partition intelligently: Too few = slow, too many = overhead

Case Study: Outbreak Investigation

Scenario

A foodborne illness outbreak has been reported. You have a dataset with 50,000 records from multiple restaurants, including customer demographics, food items consumed, and illness status.

Data Loading and Initial Exploration

# Load outbreak data
outbreak_df = spark.read.csv("outbreak_data.csv", header=True, inferSchema=True)

# Check data quality
print(f"Total records: {outbreak_df.count()}")
outbreak_df.printSchema()

# Initial summary
outbreak_df.groupBy("ill").count().show()
outbreak_df.groupBy("restaurant").count().show()

Descriptive Epidemiology: Person, Place, Time

from pyspark.sql.functions import to_date, hour, dayofweek

# Parse datetime
outbreak_df = outbreak_df.withColumn(
    "meal_datetime", 
    to_timestamp("meal_datetime", "yyyy-MM-dd HH:mm:ss")
)

# Time distribution (epidemic curve)
epi_curve = outbreak_df.filter(col("ill") == 1) \
    .groupBy(to_date("meal_datetime").alias("date")) \
    .count() \
    .orderBy("date")

epi_curve.show()

# Person characteristics
outbreak_df.filter(col("ill") == 1) \
    .groupBy("age_group", "sex") \
    .count() \
    .orderBy("age_group", "sex") \
    .show()

# Place distribution
outbreak_df.filter(col("ill") == 1) \
    .groupBy("restaurant", "city") \
    .count() \
    .orderBy(desc("count")) \
    .show(10)

Analytical Epidemiology: Food-Specific Attack Rates

# Calculate attack rates for each food item
food_items = ["chicken", "salad", "pasta", "seafood", "dessert"]

attack_rates = []

for food in food_items:
    # Those who ate the food
    ate_ill = outbreak_df.filter(
        (col(food) == 1) & (col("ill") == 1)
    ).count()
    ate_total = outbreak_df.filter(col(food) == 1).count()
    
    # Those who didn't eat the food
    not_ate_ill = outbreak_df.filter(
        (col(food) == 0) & (col("ill") == 1)
    ).count()
    not_ate_total = outbreak_df.filter(col(food) == 0).count()
    
    # Calculate attack rates
    ar_ate = ate_ill / ate_total if ate_total > 0 else 0
    ar_not_ate = not_ate_ill / not_ate_total if not_ate_total > 0 else 0
    
    # Risk ratio
    rr = ar_ate / ar_not_ate if ar_not_ate > 0 else float('inf')
    
    attack_rates.append({
        "food": food,
        "ate_ill": ate_ill,
        "ate_total": ate_total,
        "ar_ate": ar_ate,
        "ar_not_ate": ar_not_ate,
        "risk_ratio": rr
    })

# Convert to DataFrame and display
ar_df = spark.createDataFrame(attack_rates)
ar_df.orderBy(desc("risk_ratio")).show()

Incubation Period Analysis

# Calculate time from exposure to illness onset
from pyspark.sql.functions import unix_timestamp

outbreak_df = outbreak_df.withColumn(
    "incubation_hours",
    (unix_timestamp("symptom_onset") - unix_timestamp("meal_datetime")) / 3600
)

# Incubation period distribution
outbreak_df.filter(col("ill") == 1) \
    .select(
        mean("incubation_hours").alias("mean_hours"),
        expr("percentile_approx(incubation_hours, 0.5)").alias("median_hours"),
        min("incubation_hours").alias("min_hours"),
        max("incubation_hours").alias("max_hours")
    ).show()

# This helps identify the pathogen (e.g., Salmonella: 6-72h, Staph aureus: 1-6h)
📊 Key Findings In this outbreak investigation, PySpark enabled rapid analysis of 50,000 records to identify:
  • The implicated food item (highest risk ratio)
  • Time distribution suggesting point-source outbreak
  • Incubation period consistent with specific pathogen
  • Geographic clustering to specific restaurant locations
